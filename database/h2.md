## H2
H2 데이터베이스는 자바로 작성된 경량 관계형 데이터베이스 관리 시스템(RDBMS)으로 오픈 소스로 제공되며, 뛰어난 성능과 작은 용량이 특징이다.
클라이언트-서버 모드와 임베디드 모드 두 가지 모드로 실행할 수 있고 C-S 모드에서는 독립된 H2 서버를 실행하고, 
다른 애플리케이션은 해당 서버에 연결하여 데이터베이스에 접근할 수 있다.
H2 데이터베이스는 Java에서 사용하기에 편리하며, JDBC(Java Database Connectivity)를 통해 다른 프로그래밍 언어와의 연동도 가능하며, 
또한 H2는 별도의 설치 과정이 필요하지 않고 JAR 파일 형태로 제공되므로 사용하기가 간편하다.

### 주 용도
- **테스트·로컬 개발용 인메모리 DB**로 가장 많이 쓰인다. 애플리케이션이 뜰 때 메모리에 DB를 생성하고 종료 시 사라지므로, 통합 테스트를 외부 DB 없이 빠르고 격리되게 돌릴 수 있다.
- 운영 환경의 주 DB로는 거의 쓰지 않으며(인메모리는 재시작 시 데이터 소멸), 운영 DB(MySQL 등)와 **방언(SQL dialect)이 달라** H2에선 통과한 쿼리가 운영에서 깨질 수 있다 → 아래 MySQL 호환 모드로 완화한다.

### In-Memory
1. gradle 의존성 선언 
2. application.yml 설정 내용 추가 및 실행
```yml
spring:
  datasource:
    url: jdbc:h2:mem:testdb;NON_KEYWORDS=USER   # USER 등 H2 예약어를 컬럼/테이블명으로 쓰기 위해 예약어에서 제외
    username: sa
    password:
    driver-class-name: org.h2.Driver
```

### Remote
1. H2 다운로드: [https://www.h2database.com](https://www.h2database.com/html/main.html)
2. 다운로드 폴더 이동, 권한 부여 및 실행
   - 루트 기준 : chmod 755 h2/bin/h2.sh
3. localhost:8082/h2 접근 및 db 파일 생성
   - {filename}.mv.db 파일 생성 
     - JDBC URL: jdbc:h2:~/{filename} 입력 후 연결 시 파일 자동생성 진행 :: **filename 개별 작성**
   - 생성된 파일 확인 및 접근 주소 변경
     - JDBC URL: jdbc:h2:tcp://localhost/~/{filename}
4. application.yml 설정 내용 추가 및 실행
```yml
spring:
  datasource:
    url: jdbc:h2:tcp://localhost/~/{filename}
    username: sa
    password:
    driver-class-name: org.h2.Driver
```

### H2 웹 콘솔 & MySQL 호환 모드
- **웹 콘솔**: 브라우저에서 DB를 조회·실행할 수 있는 콘솔을 켠다.
```yml
spring:
  h2:
    console:
      enabled: true        # http://localhost:8080/h2-console 로 접속
      path: /h2-console
```
- **MySQL 호환 모드**: 테스트는 H2, 운영은 MySQL일 때 방언 차이를 줄이려면 접속 URL에 `MODE=MySQL`을 준다.
```
jdbc:h2:mem:testdb;MODE=MySQL;DATABASE_TO_LOWER=TRUE
```
  - 단, 완전한 호환은 아니므로 가능하면 **테스트도 운영과 같은 DB(Testcontainers의 MySQL 등)** 로 검증하는 것이 가장 안전하다.
